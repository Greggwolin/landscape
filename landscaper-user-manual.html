Landscaper User Manual

How Landscape's AI assistant adapts to what you're working on

Overview

Landscaper is an AI assistant embedded in the left panel of every project page in Landscape. It's aware of the page you're on, the data in your project, and the documents you've uploaded. It can read and write data, run calculations, extract information from documents, and manage what-if scenarios.

Landscaper uses a Level 2 Autonomy model: it can read anything freely, but all data changes are proposed as "mutations" that you must approve or reject before they take effect.

UI Layout

The Landscaper panel has four zones from top to bottom:

ZoneDescription
Header BarShows "Landscaper" label, page context badge, New Chat (+) button, and thread list toggle
Thread TitleShows current conversation title. Click to edit. Auto-generates after your first exchange.
MessagesScrollable conversation. User messages on right, Landscaper on left. Mutation proposals appear as confirm/reject cards.
InputText input with Send button. Enter sends, Shift+Enter for new line.
Context Badge: The pill in the header (e.g., "Property", "Valuation · DCF") tells you what focus mode Landscaper is in. This changes automatically as you navigate tabs.

Chat Threads

Each conversation with Landscaper is a separate thread. Threads are scoped to the project, not to a specific page — so you can start a thread on the Property tab and continue it on Valuation.

  • New Thread: Click the + button in the header
  • Switch Threads: Click the menu icon (⋮) to see your thread list
  • Rename Thread: Click the title bar above messages, or click the pencil icon
  • Auto-Naming: After your first exchange, Landscaper generates a short title from the topic discussed

Thread history is preserved — when you reopen a thread, Landscaper has full context of what was discussed.

Mutations & Approvals

When Landscaper needs to change data (update a rent, add a unit, modify an expense), it proposes the change as a mutation. You'll see a card showing exactly what will change, with Confirm and Reject buttons.

  • Read operations (queries, lookups, calculations) execute immediately — no approval needed
  • Write operations (updates, creates, deletes) always require your confirmation
  • Batch mutations group multiple related changes into one approval

Page-Aware Focus

Landscaper adapts its capabilities based on the page you're viewing. On the Property tab, it has tools for units, leases, and rent roll. On Valuation, it has cash flow and comparable tools. This focus keeps responses fast and relevant — it's not trying to process 135 tools when you only need 20.

The system works in four tiers:

Tool Tiers

TierToolsWhen Available
Universal Project fields, field schema, document read Always — every page
Page-Specific Varies (units, leases, comps, budget, etc.) Only on the matching page/tab
Extraction Document ingestion, column mapping, rent roll delta Documents page, or when you mention "document", "upload", "rent roll", etc.
What-If Scenario compute, compare, commit, undo Valuation, Capitalization, Reports, and IC pages
Universal tools include get_project_fields (read any project field), get_project_documents and get_document_content (search uploaded docs). These are always available regardless of page, so Landscaper can always look up basic project data and search your documents.

Data Lookup Priority

When you ask Landscaper about property data, it follows this order:

#SourceAction
1DatabaseQueries project fields, units, leases, etc. using available tools
2Uploaded DocumentsSearches OMs, appraisals, T-12s automatically — no permission needed
3Ask YouOnly if neither the database nor documents have the answer

Landscaper should never say "I don't have that data, would you like me to search documents?" — it searches automatically. Financial assumptions (cap rate, discount rate, growth rates) always come from the database, never from documents.

What-If Scenarios

Available on Valuation, Capitalization, Reports, and IC pages. Scenarios let you test assumptions without changing actual data.

ToolWhat It Does
whatif_computeTest a single assumption change and see the impact
whatif_compoundStack multiple changes together
whatif_resetClear the active what-if state
scenario_saveSave current what-if as a named scenario
scenario_loadLoad a previously saved scenario
scenario_compareSide-by-side comparison of scenarios
scenario_diffShow delta between two scenarios

whatif_undoReverse the last committed change
sensitivity_gridMatrix of outcomes across two variables

Tips & Best Practices

  • Be direct. "What's the NOI?" works better than "Can you tell me what the net operating income is for this property?"
  • One question at a time gets better results than compound questions.
  • Name your numbers. "Set cap rate to 5.25%" is clearer than "lower the cap rate a bit."
  • Use the right page. If Landscaper can't do something, you might be on the wrong tab. Switch to the relevant page and try again.
  • Mention "document" if you want to pull data from uploaded files while on a non-Documents page.
  • What-if before committing. Test scenarios on Valuation before updating actual data.
  • Review mutations carefully. Landscaper shows exactly what will change — read the diff before confirming.
