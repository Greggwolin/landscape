The first slice, in one paragraph

Slice 1 — "A land deal is born on the map." A workspace map that belongs to the app rather than any project. A new deal starts by selecting county tax parcels or drawing a boundary; that act creates the project with geometry, acreage, and parcel identity pre-filled. Inside the project, the parcel table is authored spatially — multi-select or paint on the plan, attributes validated by the canonical land use library, grouped/flat table views in sync — with the docket handling every judgment call. Pilot: recreate Red Valley Ranch from scratch in it.

What happens next, in order

  1. The locked register goes to the coding side as a briefing — along with the instruction that their first move is an audit of the existing map, overlay, and parcel machinery, so the slice is specified against what actually exists rather than what anyone remembers.
  2. The coding side returns a phased implementation plan with explicit approval gates. Nothing is built before you approve the phasing.
  3. Slice 1 ships to you; you run Red Valley Ranch through it for real. What the workflow demands next — streets, intake, outputs — gets chosen from that experience, not from a roadmap written in the dark.
