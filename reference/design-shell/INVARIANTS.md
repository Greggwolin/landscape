# INVARIANTS — conditional behaviors to preserve in the `design` wrapper
_Running log. Each entry = a rule where UI behaves differently based on data state. The `design` shell reuses these components as-is; do not reimplement the logic._

## Property
- **FloorPlanMatrix** (`property/FloorPlanMatrix.tsx`): source hierarchy keyed on **active/month-to-month leases**. Active leases → aggregate from rent roll, badge "From Rent Roll", **read-only**. None → `tbl_multifamily_unit_type`, badge "From Unit Types", **editable**. Not keyed on mere rent-roll presence.
- **PhysicalDescription** (`property/PhysicalDescription.tsx`): 42 fields / 6 sections; click-to-edit inline; ratings as ★ dropdown; lot-size acres↔SF auto-convert (1 ac = 43,560 SF); per-section fill progress toggles show-all vs populated-only; extracted values carry provenance lock.
- **FloorplansGrid** (`rent-roll/components/FloorplansGrid.tsx`): AG-Grid unit-type master; every cell editable with **auto-save on cell change** (toast; reverts cell on failure); add (NEW-n) / delete (confirm) rows; **auto-refreshes on Landscaper unit_types/units mutations** (`useLandscaperRefresh`).
- **FloorplanUpdateDialog** (`rent-roll/components/FloorplanUpdateDialog.tsx`): when entered specs diverge from a doc-backed master floorplan, prompt three ways — update this unit only · update the master floorplan (affects N units, drives auto-fill) · create a new floorplan (needs a new code). Only offers "update existing" when a floorplan record exists.
- **RenovationSubTab / ValueAddCard** (`tabs/RenovationSubTab.tsx`, `operations/ValueAddCard.tsx`): sub-tab present **only when value_add_enabled**; **empty state when totalUnits === 0** ("add unit types in Rent Roll/Operations first"). Renovation cost carries a **/SF ↔ /Unit basis toggle**; units-to-renovate = All or a **custom count clamped to totalUnits**; costs/annual-impact/treemap are **derived (not editable)**; assumptions auto-save; all figures drive the Operations **Post-Reno** column.
- **ProjectLandUseLabels** (`project/ProjectLandUseLabels.tsx`): the 6 label fields are **project-scoped** and re-term the level-1/2/3 hierarchy (Area/Phase/Parcel by default) across **planning grids, the inventory tree, and reports** — `useProjectConfig` reads `tier_1/2/3_label` with those fallbacks and `level1_enabled`/`level2_enabled` gating. Save/Reset enable only when dirty.
- **ProjectDates** (`project/ProjectDates.tsx`): `analysis_start_date` is **T0** — period index 1 = that date; absolute dates everywhere derive from it. Save enables only when dirty.
- General: forms with dirty-state Save (dates, labels, profile) — the button is disabled until a field changes; profile Details is a read-only rollup, edited only via the modal.
